# REQUIREMENTS — 어스얼라이언스 백엔드 엔지니어 채용 과제

## 개요
NestJS로 작업(Job)을 관리하는 백엔드 시스템을 개발한다. RESTful API로 작업을 생성·조회·검색하고,
백그라운드 작업으로 작업을 주기적으로 처리한다. 데이터는 단일 JSON 파일에 영속화한다.

> 기능 구현뿐 아니라 **API 설계 결정**과 **여러 컴포넌트가 같은 데이터를 다루는 환경에서의 처리**를
> 직접 판단하여 수행한다.

## 기술 스택
- 프레임워크: **NestJS (TypeScript)**
- 데이터 저장: **[node-json-db](https://www.npmjs.com/package/node-json-db)** (단일 JSON 파일)
- 스케줄러: **@nestjs/schedule**
- 그 외 라이브러리는 자유

## 데이터 스키마
자유 설계 가능. 최소 다음을 표현할 수 있어야 한다.
```json
{ "jobs": [ { "id": "uuid", "title": "Task 1", "description": "Do something", "status": "pending" } ] }
```

## API 명세
| Method | Path | 설명 |
| --- | --- | --- |
| POST | `/jobs` | 새 작업 생성 |
| GET | `/jobs` | 작업 목록 조회 |
| GET | `/jobs/search` | 제목/상태로 검색 |
| GET | `/jobs/:id` | 단일 작업 조회 |
| PATCH | `/jobs/:id` | 작업 수정 |

- 응답 포맷, 검색 쿼리 파라미터, `PATCH` 가능 필드와 **상태 전이 규칙**, 에러 응답 구조는 자유 설계.
- HTTP 시맨틱에 맞는 상태 코드와 **일관된 에러 응답**.
- 모든 요청을 `logs.txt`에 로깅.

## 백그라운드 스케줄러
- 생성된 작업은 백그라운드에서 **일정 주기로 상태를 점검하여 처리**한다.
- 처리 주기와 한 번에 처리할 단위는 자유롭게 가정하여 결정한다 (예: 1분 간격).
- 처리 결과를 `logs.txt`에 로깅.

## 동시성
- API 요청과 스케줄러가 **동시에 같은 데이터에 접근**할 수 있는 환경.
- 동시 요청 상황에서도 데이터가 손실되거나 깨지지 않도록 고려한다.
